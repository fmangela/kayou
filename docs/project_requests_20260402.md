# 卡牌属性设置技术文档

# 1. 文档说明

## 1.1 文档目的

本文档用于规范卡牌属性的设计标准、字段定义及关联规则，为卡牌开发、数据录入、系统对接提供统一依据，确保卡牌属性设置的一致性、规范性和可扩展性，适配多系列、多阵营卡牌的迭代更新。

## 1.2 适用范围

本规范适用于所有卡牌的属性设计、技能配置、数据维护及相关开发工作，涵盖三国系列、奥特曼系列等所有已规划及后续新增系列卡牌，包括基础属性、技能属性、隐藏属性的全流程定义。

## 1.3 核心原则

- 规范性：所有卡牌属性字段、格式、取值范围统一，避免歧义；

- 可扩展性：预留属性扩展空间，适配后续新增系列、阵营及属性类型；

- 关联性：明确卡牌属性与技能表、隐藏关系技能的关联逻辑，确保数据一致性；

- 可读性：字段定义清晰，示例明确，便于开发、运营及数据人员理解和操作。

# 2. 卡牌基础属性定义

卡牌基础属性为所有卡牌的必选字段，用于唯一标识卡牌、区分卡牌归属及核心基础能力，各字段定义如下：

## 2.1 卡牌id号

- 定义：卡牌的唯一标识符，用于系统内卡牌数据的检索、存储及关联，不可重复；

- 格式：系列拼音缩写大写字母-弹数-4位编号；

- 取值范围：新增卡牌按序号依次递增；

- 备注：系列编码、阵营编码需提前规划并同步至《卡牌编码规范表》，新增系列/阵营需更新编码表。

- 实例ID号：玩家获得卡牌后的实例ID号，使用mysql自增ID

## 2.2 名称

- 定义：卡牌的直观名称，用于展示给用户，需贴合卡牌所属系列及人物/角色特征；

- 格式：中文名称，长度控制在2-8个字，避免生僻字、歧义字；

- 示例：三国系列蜀国卡牌“诸葛亮”，奥特曼系列卡牌“迪迦奥特曼”，怪兽系列卡牌“哥尔赞”。

## 2.3 系列

- 定义：卡牌所属的核心系列，用于区分不同主题的卡牌集合；

- 取值范围：已规划系列（三国系列、奥特曼系列），后续新增系列需在文档中同步更新；



## 2.4 阵营

- 定义：卡牌在对应系列中的归属阵营，用于区分同系列内的不同势力/群体；

- 取值规则：
        

    - 举例三国系列：蜀国、魏国、吴国、诸侯国；

    - 举例奥特曼系列：奥特曼、怪兽、宇宙人、中立；

    - 后续新增系列需同步补充对应阵营，中立阵营适用于无明确势力归属的卡牌。

## 2.5 属性

- 定义：卡牌的核心元素/能力类型，用于影响卡牌间的克制关系、技能效果触发；

- 取值范围：属性（金、木、水、火、土）、属性（格斗、光线、技巧），可根据新增系列补充属性；

- 备注：单张卡牌仅可拥有1个属性，不可多属性叠加，属性克制关系需在后续《卡牌克制规则文档》中明确。

## 2.6 六维属性

- 定义：卡牌的核心能力量化指标，统一为5项，用于衡量卡牌的战斗能力、生存能力及辅助能力；

- 具体指标及定义：
        

    - 武力：卡牌的物理攻击能力，影响物理伤害输出；

    - 智慧：卡牌的技能释放效率、策略能力，影响技能伤害、控制效果命中率；

    - 防御：卡牌的物理/法术防御能力，影响受到的伤害减免效果；

    - 速度：卡牌的出手顺序，速度值越高，出手优先级越高；

    - 精神：卡牌的抗控制能力、技能续航能力，影响控制效果抵抗率、技能冷却缩减；

    - 体力：卡牌的血量能力，体力越高，血量越高。

- 数值范围：六维属性之和与稀有度匹配；

- 备注：同稀有度卡牌的三维数值可根据卡牌定位（攻击型、防御型、辅助型）差异化分配，避免数值同质化。

## 2.7 稀有度

- 定义：卡牌的稀缺程度，用于区分卡牌的获取难度、能力上限；

- 取值范围（按稀有度从低到高排序）：N（普通）、R（稀有）、SR（超级稀有）、SSR（特级稀有）、UR（终极稀有）、PR（限定稀有）、HR（全息稀有）；

- 标识规范：每张卡牌需在名称旁标注稀有度标识，PR/HR级为限定卡牌，仅可通过特定活动获取，不可通过常规卡池抽取。

## 2.8 人物背景描述

- 定义：卡牌对应人物/角色的背景介绍，用于丰富卡牌故事性，提升用户体验；

- 格式：中文文本，长度控制在50-200字，贴合系列背景、人物设定，语言简洁易懂；

- 示例：诸葛亮（三国系列蜀国）：字孔明，号卧龙，三国时期蜀汉丞相，足智多谋，善用谋略，为蜀汉基业立下汗马功劳，是智慧的象征。

# 4. 隐藏属性（关系技能）

## 4.1 定义

隐藏属性特指卡牌的关系技能，即当满足特定卡牌组合条件时，触发的额外加成或特殊效果，不单独显示在卡牌表面，仅在满足条件时生效，用于提升卡牌组合的策略性。

## 4.2 触发条件

- 同系列组合：多张同系列卡牌同时在场（如3张三国系列卡牌）；

- 同阵营组合：多张同阵营卡牌同时在场（如2张蜀国卡牌+1张魏国卡牌，不触发同阵营组合，需全部同阵营）；

- 特定人物组合：指定的2-3张卡牌同时在场（如诸葛亮+刘备+关羽，触发“蜀汉三杰”关系技能）；

- 属性组合：多张特定属性的卡牌同时在场（如金、木、水、火、土属性各1张，触发“五行归一”关系技能）。

## 4.3 效果类型

- 属性加成：触发组合的所有卡牌三维属性提升（如全体武力+10%、智慧+15%）；

- 技能强化：触发组合的特定卡牌技能效果提升（如诸葛亮技能冷却缩减2回合）；

- 特殊效果：触发额外的群体伤害、治疗、控制等效果（如“蜀汉三杰”组合触发“桃园结义”，全体回血20%）；

- 隐藏技能解锁：触发组合后，解锁某张卡牌的隐藏主动技能（该技能不显示在常规技能列表中）。

## 4.4 关联规则

- 关系技能需与卡牌id关联，明确触发所需的卡牌组合（列出所有关联卡牌id）；

- 单张卡牌可参与多个关系技能（如诸葛亮可参与“蜀汉三杰”“三国谋士”等多个组合）；

- 关系技能效果可叠加（如同时满足同系列+同阵营组合，两个组合的加成效果同时生效）；

- 所有关系技能需单独记录在《卡牌关系技能表》中，与卡牌id、技能表关联，便于系统检索触发。

# 5. 数据关联规则

- 卡牌基础属性与技能表：通过“卡牌id号”关联，一个卡牌id对应1-3个技能id；

- 卡牌基础属性与关系技能：通过“卡牌id号”关联，一个卡牌id可关联多个关系技能，一个关系技能可关联多个卡牌id；

- 技能表与关系技能：关系技能可强化技能表中的技能效果，通过“技能id号”关联；

- 新增、修改、删除卡牌时，需同步更新技能表、关系技能表中对应的关联数据，确保数据一致性。

# 7. 后台管理功能

## 7.1 卡牌属性修改功能

- 功能定义：后台管理系统需具备卡牌全属性修改权限，用于卡牌数据调整、错误修正及版本迭代时的属性更新，确保卡牌数据的灵活性和准确性。

- 修改范围：可修改卡牌基础属性（除卡牌id号外，名称、系列、阵营、属性、六维属性、稀有度、人物背景描述）、技能表关联技能信息、隐藏关系技能触发条件及效果。

- 操作规则：
                  修改操作需记录操作日志（操作人、操作时间、修改前内容、修改后内容），便于追溯；

- 卡牌id号不可修改，避免破坏与技能表、关系技能表的关联逻辑；

- 修改后需实时同步至相关关联表格，确保数据一致性，修改完成后需有确认弹窗，防止误操作。

权限控制：仅管理员及指定权限人员可操作，普通用户无修改权限，不同权限人员可修改的属性范围可按需配置。

## 7.2 批量导入导出功能

- 功能定义：支持卡牌数据（含基础属性、技能信息）的批量导入与导出，提升数据维护效率，适配大量卡牌新增、更新及备份需求。

- 批量导出：
                  导出格式：支持Excel、CSV两种格式，导出内容可按需选择（全部卡牌数据、指定系列/阵营/稀有度卡牌数据、仅基础属性、含技能关联数据）；

- 导出规则：导出文件需包含字段说明，确保导出数据可直接用于查看、备份或二次编辑，导出过程中需显示进度提示，导出完成后可下载保存。

批量导入：
                  导入格式：仅支持Excel、CSV格式，导入文件需严格遵循预设模板（模板可从后台下载），字段顺序、格式需与模板一致；

校验规则：导入前系统自动校验数据（卡牌id唯一性、字段格式正确性、数值范围合规性、技能与卡牌关联有效性），校验失败会提示错误信息（错误位置、错误原因），可修改后重新导入；

导入逻辑：导入数据可选择“覆盖原有数据”“新增数据”“跳过重复数据”三种模式，导入完成后生成导入报告，明确导入成功数量、失败数量及失败原因。

备注：批量导入导出操作同样需记录操作日志，导入敏感数据（如稀有度、数值）需额外验证操作权限，防止数据泄露或误操作。
