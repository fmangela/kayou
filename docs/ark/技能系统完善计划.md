## 完善路线图

### 第一阶段：核心对战接入（优先级最高）

目标：让技能系统实际在对战中生效

1. **实现效果原子运行时逻辑** `backend/services/skillEngineService.js`
   - `modify_stat_flat` - 属性数值修改
   - `modify_stat_ratio` - 属性倍率修改
   - `add_status` - 添加状态效果
   - `remove_status` - 移除状态效果
   - `apply_value_clamp` - 数值钳制
   - `grant_extra_trigger` - 额外触发机会
   - `reduce_cooldown` - 减少冷却
   - `modify_damage_flat` - 伤害数值补正
   - `modify_damage_ratio` - 伤害倍率补正
   - `apply_trigger_limit` - 触发次数限制

2. **在对战流程中集成技能引擎** `frontend/src/views/admin/BattleTestView.vue`
   - 战斗开始前，加载所有出战卡牌技能
   - 根据触发事件顺序应用技能效果
   - 处理触发事件：开局、进攻前、进攻后、结算前、结算后
   - 记录状态效果，战斗中持续生效

3. **修改对战伤害计算**，应用技能修改后的属性

### 第二阶段：卡牌制作整合

目标：在卡牌制作流程中可以选择结构化技能

1. **卡牌表结构修改**
   - 当前 `card_attributes` 已有 `skill1_id/skill2_id/skill3_id` 字段
   - 需要改成关联 `skill_definitions.id` 或 `skill_uid`
   - 卡牌展示时，从技能定义读取名称和描述，不再冗余存储

2. **CardMakerView.vue 修改**
   - 添加技能选择弹窗
   - 可以从已发布技能列表搜索选择
   - 选择后自动填充名称和描述到属性表
   - 支持清空重新选择

3. **CardPreview.vue 修改**
   - 卡牌渲染时从技能定义读取描述
   - 确保文字位置大小正确

### 第三阶段：前端体验优化

改善管理员操作体验

1. **模板编辑体验改进**
   - 当前参数槽需要手写 JSON，改为可视化添加
   - 每个参数一行，可以增删改
   - 自动生成 JSON，不需要用户懂 JSON

2. **功能补充**
   - 添加 **复制技能** 功能（基于已有技能新建）
   - 添加 **软删除** 功能（标记为归档）
   - 添加 **分页加载**（技能多了之后不卡）
   - 添加更多筛选条件：按技能类型、标签筛选
   - 支持按强度预算范围筛选

3. **优化表格展示**
   - 添加触发事件和效果原子列
   - 可以按类型排序

### 第四阶段：数据迁移

把现有手工填写的技能转换成结构化技能

1. **迁移脚本**
   - 读取现有卡牌上的技能名称和描述
   - 调用大模型，自动识别匹配模板+填充参数
   - 生成结构化技能定义
   - 人工审核确认

2. **更新卡牌关联**
   - 迁移完成后，更新卡牌表的 skill_id 关联
   - 保持向后兼容

### 第五阶段：高级功能完善

1. **扩展效果原子库**，支持更多效果类型：
   - 概率触发
   - 条件满足才触发
   - 对特定目标（比如队长/同阵营/不同稀有度）生效
   - 累计叠加效果
   - 回合结束/开始触发
   - 阵亡触发遗言
   - 等等

2. **条件表达式支持**
   - 技能可以设置触发条件
   - 基于卡牌状态、战局状态判断

3. **平衡性管理**
   - 根据模板强度预算检查技能强度
   - 统计分析不同稀有度的技能强度分布
   - 给出调整建议

4. **导入导出**
   - 支持 JSON 导入导出技能
   - 方便分享和备份

## 当前数据库结构

### skill_templates（技能模板）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | int | PK |
| template_key | varchar | 唯一键 |
| name | varchar | 模板名称 |
| skill_type | varchar | 技能类型 |
| trigger_domain | varchar | 触发域 |
| effect_atom | varchar | 效果原子 |
| param_slots | json | 参数槽定义 |
| description_template | text | 描述模板 |
| applicable_rarities | json | 适用稀有度 |
| strength_budget | json | 强度预算 |
| status | varchar | 状态 |

### skill_definitions（技能定义）

| 字段 | 类型 | 说明 |
|------|------|------|
| id | int | PK |
| skill_uid | varchar | 稳定唯一 ID |
| version | int | 版本号 |
| name | varchar | 技能名 |
| short_desc | varchar | 简述 |
| long_desc | text | 详细描述 |
| template_id | int | FK 关联模板 |
| params | json | 参数值 |
| skill_type | varchar | 技能类型 |
| trigger_event | varchar | 触发事件 |
| effect_atom | varchar | 效果原子 |
| target | varchar | 目标范围 |
| tags | json | 标签 |
| applicable_rarities | json | 适用稀有度 |
| slot_semantic | varchar | 槽位语义 (main/sub/awakening) |
| source | varchar | 来源 |
| status | varchar | 状态 |
