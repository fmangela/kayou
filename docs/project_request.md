# 卡牌网页游戏功能需求文档

## 一、文档概述

### 1.1 项目背景

本项目为**2D回合制卡牌网页游戏**，主打轻量休闲玩法，核心以玩家自选4张卡牌出战，通过回合制完成小游戏计分结算伤害，以随机单人爬塔闯关为核心玩法，附带轻度玩家对战功能；采用Vue3+PixiJS+JS+CSS+2D Canvas+MySQL技术栈开发，优先适配手机端网页浏览，兼顾PC端网页体验，本次需求文档聚焦游戏入口及核心功能模块的基础需求，为前期开发提供明确依据。

### 1.2 核心玩法重申

玩家提前选择4张卡牌组成出战卡组，每回合触发随机轻量小游戏，根据小游戏表现计算基础分数，结合卡牌属性/技能倍率结算伤害，用于对抗关卡怪物或对战对手；核心玩法为随机性单人爬塔闯关，额外开放玩家PVP对战功能，整体操作简洁，适配手机端触摸交互，网页加载轻量化。

### 1.3 技术栈适配（暂定）

- 前端框架：Vue 3 (Composition API) + TypeScript

- 渲染引擎：PixiJS（负责卡牌、战场及高级特效渲染） + Canvas 2D（负责轻量回合制小游戏开发）

- UI 组件库：Element Plus（用于非战斗界面的管理面板、表单及商店交互）

- 状态管理：Pinia（管理玩家属性、背包卡组及全局加载状态）

- 后端支撑：Node.js + Express

- 实时通讯：Socket.IO（用于 PVP 对战同步及实时奖励通知）

- 持久化层：MySQL（存储用户信息、卡牌数据及关卡进度）

- 核心要求：手机端网页优先，响应式适配，操作轻量化，无冗余加载

## 二、游戏入口与登录模块需求

### 2.1 入口界面

1. 页面定位：游戏唯一初始访问页面，简洁轻量化，无复杂动画，优先适配手机端竖屏

2. 页面元素：游戏名称（醒目2D文字+简单卡牌背景底图）、登录入口按钮、极简版本号标注、隐私政策简易提示

3. 交互逻辑：点击登录按钮直接跳转登录表单页面，无其他冗余跳转，页面加载时长≤3秒

### 2.2 登录功能（极简版）

1. 登录方式：仅支持**用户名+密码**登录，暂不开发验证码、第三方登录、手机号登录等复杂功能

2. 表单元素：用户名输入框、密码输入框、登录确认按钮、注册新用户入口按钮

3. 注册逻辑：

    - 新用户点击注册，弹出极简注册表单，仅需填写用户名、密码、确认密码，无额外信息填写

    - 用户名唯一，后端MySQL数据库校验重复，密码支持数字+字母组合，无强制复杂度要求

    - 注册成功直接跳转登录页面，自动填充用户名，无需二次验证

4. 登录校验：

    - 前端基础校验：用户名、密码非空判断，输入为空时弹出文字提示

    - 后端校验：核对MySQL数据库中用户账号密码，匹配成功进入游戏主界面；失败弹出“用户名或密码错误”提示

    - 登录状态：登录成功后保留本地登录状态，短时间内重新进入游戏无需重复登录，退出登录后清除状态

5. 异常处理：网络异常时弹出“网络连接失败，请重试”提示，禁止跳转

## 三、游戏主界面需求

登录成功后直接进入游戏主界面，主界面为功能入口聚合页，采用简洁2D布局，PixiJS渲染基础视觉元素，手机端竖屏适配，核心功能入口清晰排布，无复杂视觉干扰；顶部展示玩家基础信息，中部为核心功能图标入口，底部为简易导航栏。

## 四、核心功能模块详细需求

### 4.1 人物状态模块

1. 模块定位：展示玩家基础游戏信息，核心数据可视化，无复杂操作，点击主界面玩家头像即可进入

2. 展示内容：

    - 基础信息：玩家用户名、当前等级、经验值（升级进度条）、体力值（爬塔/对战消耗资源）、金币（基础货币）、钻石（稀有货币）、卡牌粉尘（用于提升稀有卡牌获取概率）

    - 战斗属性：总战力（基于出战卡牌计算）、当前出战卡组（4张卡牌缩略图+核心属性）、闯关进度（当前爬塔层数）、对战胜负场数

    - 成就统计：基础成就（首次通关、对战胜利、卡牌收集数量）完成情况

3. 交互逻辑：

    - 点击体力/金币/钻石，弹出资源获取途径提示

    - 点击出战卡组卡牌，可快速跳转卡牌包模块调整卡组

    - 支持等级升级弹窗提示，展示升级奖励与属性提升

4. 数据同步：所有数据实时从MySQL数据库拉取，修改后自动同步后端

### 4.2 卡牌包模块

1. 模块定位：玩家卡牌管理核心模块，核心功能为卡牌查看、筛选、出战卡组设置，是核心玩法的前置模块

2. 核心功能：

    - 卡牌展示：所有已拥有卡牌列表，PixiJS渲染卡牌2D形象，卡牌图片采用webp格式（轻量化、画质清晰，适配网页加载）；卡牌图片与文字属性分离设计，图片仅作为卡牌视觉载体，卡牌名称、等级、属性、技能、伤害倍率、稀有度、所属系列等文字及数值信息单独渲染，便于后期通过后台修改卡牌属性，无需重新替换卡牌图片。

    - 卡牌按系列划分，每个卡牌归属对应系列，展示时标注卡牌所属系列标识

    - 筛选排序：支持按稀有度、等级、属性、获取时间筛选排序，方便玩家快速查找卡牌

    - 卡组设置：核心规则为**仅可选择4张卡牌组成出战卡组**，已选卡牌标醒目提示，超出4张时无法选择，弹出“最多选择4张出战卡牌”提示

    - 卡牌操作：单张卡牌点击查看详情；

    - 未拥有卡牌：灰色占位展示，标注“未获取”，可跳转图鉴查看完整信息

    - 卡牌获取与重复卡牌逻辑：当前卡牌仅可通过商店购买卡牌包并拆包获取，无其他获取途径；拆包过程中，若抽到已拥有的重复卡牌，该重复卡牌将自动转换为卡牌粉尘；卡牌粉尘的核心用途为提高稀有卡牌获取概率，玩家可在商店卡牌包拆包前消耗对应数量的卡牌粉尘，提升本次拆包抽到稀有卡牌的几率，暂不开放其他用途。

3. 交互逻辑：

    - 卡牌选择支持触摸点击/短按选中，取消选中再次点击即可，没有拖拽

    - 卡组设置完成后点击“保存出战卡组”按钮，数据同步至后端MySQL，闯关/对战默认调用该卡组

    - 新增卡牌组合预设功能，默认开放3组卡牌组合预设位置，玩家可给每组预设命名（如“爬塔专用”“对战专用”），方便快速切换出战卡组；后期可通过消耗金币购买额外组合预设位置，购买后实时解锁，支持无限叠加（根据金币消耗逐步开放）。

    - 手机端适配：卡牌列表竖向滑动，单张卡牌详情弹窗适配竖屏，操作按钮居中

4.3 活动广场模块

1. 模块定位：日常福利与限时玩法入口，提升玩家留存，前期开发基础日常活动

2. 基础活动内容：

    - 每日签到：每日登录签到领取金币，连续签到额外奖励，日历形式展示签到状态

    - 日常任务：完成每日任务领取奖励，任务每日0点重置

    - 限时活动：前期开放基础爬塔双倍经验、金币翻倍活动，后期可扩展节日活动，界面标注活动时间与规则

3. 交互逻辑：

    - 奖励可直接领取，领取后实时到账，弹出奖励获得提示

    - 活动规则点击查看文字说明，简洁易懂

    - 已完成/已领取活动标醒目状态，未完成活动标注进度

### 4.4 商店模块

1. 模块定位：游戏内资源与卡牌购买入口，轻量化设计

2. 商品分类：

    - 基础商店：使用金币购买，普通卡牌包

    - 稀有商店：金币购买，售卖高级卡牌包

    - 限购商品：每日/每周限购商品，标注限购次数，防止无限购买

3. 交互逻辑：

    - 商品点击查看详情（内容、价格、限购数），确认购买后扣除对应货币，发放卡牌包至玩家账户；玩家点击卡牌包可进行拆包操作，拆包后随机获得卡牌，抽到重复已拥有卡牌自动转换为卡牌粉尘；拆包前可选择消耗卡牌粉尘，提升稀有卡牌获取概率，粉尘消耗数量可后台配置。

    - 商店商品每日0点刷新，支持钻石手动刷新（每日限次）

    - 货币不足时弹出“货币不足，无法购买”提示

### 4.5 实验室模块

1. 模块定位：卡牌养成与玩法强化模块，区别于卡牌基础升级，主打卡牌属性进阶、技能强化

2. 核心功能：

    - 为了公平性，目前卡牌无法强化

    - 道具研究解锁，道具无法用于对战，其他单人模式都可以，道具

    - 小游戏增益：解锁小游戏专属增益（如分数加成、时间延长），消耗材料激活，非对战中生效

3. 交互逻辑：

    - 强化/激活操作一键完成，结果实时同步后端，弹出成功/失败提示

### 4.6 单人爬塔模块

1. 模块定位：核心玩法模块，随机性闯关，贴合核心玩法规则

2. 核心玩法逻辑：

    - 爬塔结构：随机爬40层塔，每10层上一个负面效果，并增加难度

    - 出战规则：默认调用卡牌包中保存的4张出战卡组，不可中途更换

    - 回合流程：每回合触发随机轻量小游戏→玩家完成小游戏→前端上传操作数据→后端计算分数→结合卡牌属性结算伤害→扣除怪物血量→怪物反击扣除玩家血量→循环至一方血量为0

    - 通关规则：玩家血量为0则闯关失败，结算；怪物血量为0则通关，进入下一层，领取层奖励

    - 随机机制：每层对手、关卡buff由后端随机生成，通过MySQL存储关卡数据与随机种子

    - 关卡有 休息、商店、普通人机对战，BOSS战，精英人机对战，选择性事件

3. 交互逻辑：

    - 进入爬塔扣除对应体力，体力不足无法挑战

    - 小游戏采用2D Canvas开发，手机端触摸操作，操作时长适配手机用户习惯

    - 伤害结算、血量变化搭配PixiJS简单动效，闯关成功/失败弹出对应提示与奖励

    - 支持中途退出爬塔，退出后当前层进度不保存，体力不返还，立即结算

### 4.7 对战模块

1. 模块定位：轻度PVP玩法模块，基于Socket.IO实现实时同步，操作简洁

2. 核心玩法逻辑：

    - 对战模式：1V1玩家实时对战，双方均使用各自4张出战卡组

    - 房间机制：玩家点击快速匹配，后端自动匹配实力相近对手，创建对战房间

    - 回合流程：双方轮流进行回合，每回合各自完成小游戏→结算伤害→攻击对手血量，Socket.IO同步双方数据与界面状态

    - 胜负规则：一方血量为0则失败，另一方获胜，结算胜负场数与对战奖励

3. 交互逻辑：

    - 匹配过程展示等待动画，匹配成功弹出对战准备提示

    - 对战过程无延迟同步，断线重连后自动恢复对战状态

    - 对战结束展示胜负界面、伤害数据、奖励，可选择再来一局或返回主界面

### 4.8 图鉴模块

1. 模块定位：全卡牌展示与收集进度查看模块，满足玩家收集欲

2. 展示内容：

    - 全卡牌图鉴：按稀有度、属性分类，展示游戏内所有卡牌的2D形象（图片采用webp格式）、完整属性、技能、获取途径；卡牌图片与文字属性分离，文字及数值信息独立渲染，后台修改卡牌属性后，图鉴可实时同步更新，无需替换卡牌图片。

    - 收集进度：已收集卡牌标彩色，未收集标灰色，顶部展示总收集率

    - 卡牌故事：每张卡牌附带简易背景故事，点击查看

3. 交互逻辑：

    - 图鉴列表竖向滑动，支持分类筛选快速查找

    - 已收集卡牌可跳转卡牌包模块操作，未收集标注获取途径（商店、爬塔、活动）

4. 适配要求：所有模块优先适配手机端竖屏，CSS响应式布局，支持不同尺寸手机屏幕，操作按钮大小适配手指触摸，无点击偏差

5. 视觉风格：统一2D卡通风格，PixiJS渲染卡牌、战场元素，无复杂3D效果，页面简洁无冗余元素

6. 数据安全：核心数据（卡牌属性、小游戏分数、伤害结算、货币数量）均由后端MySQL存储与计算，前端仅做展示与操作上传，防止作弊

7. 加载要求：单模块加载时长≤2秒，整体游戏无长时间白屏，加载过程展示极简动画

8. 操作反馈：所有点击、触摸操作均有文字/动效反馈，无操作无响应情况

## 六、后台管理系统需求

后台管理系统为独立管理端页面，采用Vue3+TypeScript+Element Plus开发，适配PC端操作，仅管理员账号可登录访问，核心用于游戏运维、数据管控、内容配置，支撑游戏日常运营与调试，杜绝普通玩家访问权限，所有操作留痕，保障数据安全可追溯。

### 6.1 管理员登录与权限控制

1. 登录规则：专属管理员登录入口，独立账号密码体系，与游戏玩家账号完全隔离，禁止共用账号，支持账号密码重置、二次登录校验。

2. 权限分级：分为超级管理员、运营管理员、数据管理员三级权限，超级管理员拥有全部操作权限，运营管理员仅可操作活动、邮箱、基础配置，数据管理员仅可查看数据与修改基础数值，权限细分防止越权操作。

3. 操作日志：所有后台增删改查操作自动记录日志，包含操作人、操作时间、操作内容、IP地址，日志永久存储至MySQL，支持日志检索与导出，便于问题排查。

### 6.2 服务器状态监控

1. 实时监控：展示前端服务、后端Node.js服务运行状态，在线玩家人数、当前在线房间数、接口请求频率、服务器CPU/内存占用率、网络延迟等核心运维数据。

2. 异常告警：服务器异常、接口报错、连接超时等情况实时标红提示，支持简易服务重启触发（超级管理员权限），展示历史服务运行记录与异常排查记录。

3. 数据统计：每日/每周游戏核心数据汇总，包括新增玩家数、活跃玩家数、爬塔通关数据、对战场次、货币消耗与产出数据，支持数据导出为Excel表格。

### 6.3 玩家账号管理

1. 账号查询：支持按玩家用户名、用户ID、注册时间精准检索玩家账号，展示玩家完整信息（等级、体力、货币、卡组、爬塔进度、对战数据、封禁状态）。

2. 数值修改：可手动修改玩家各类核心数值，包括等级、经验、体力、金币、钻石、卡牌数量、战力、卡牌粉尘数量，修改前需填写修改原因，确认后同步至MySQL数据库，实时生效至玩家端。

3. 账号设置：支持玩家账号封禁/解封（禁止登录游戏）、重置密码、清空异常数据、补发丢失奖励、调整出战卡组等操作，针对违规玩家可做禁言、限制对战权限处理。

4. 批量操作：支持批量查询符合条件的玩家账号，批量发放基础奖励，提升运维效率。

### 6.4 活动管理

1. 活动配置：新增、编辑、下架游戏内各类活动（每日签到、日常任务、限时活动），配置活动名称、活动时间、活动规则、奖励内容、参与条件、活动开关状态。

2. 活动上下线：支持手动控制活动上线/下线，设置活动预热时间、结束时间，过期活动自动下架，未开启活动隐藏不展示至玩家端。

3. 奖励调整：可修改活动奖励内容、发放数量、领取限制，实时同步至游戏活动广场模块，支持紧急补发活动奖励至玩家账号。

### 6.5 邮箱系统管理

1. 邮件发送：支持单发邮件、批量全服邮件、定向分组邮件，编辑邮件标题、邮件内容、附件奖励（金币、钻石、卡牌、材料），设置邮件有效期。

2. 邮件管理：查看已发送邮件、未领取邮件、过期邮件，支持撤回未领取邮件、补发过期邮件，清理无效邮件数据。

3. 玩家端联动：后台发送的邮件实时推送至玩家游戏内邮箱入口，玩家领取后同步扣除附件，邮件状态同步至后台，保障奖励发放无遗漏。

### 6.6 游戏基础数值修改系统

1. 全局数值配置：修改游戏核心全局数值，包括玩家初始血量、初始货币、体力恢复速度、体力上限、爬塔体力消耗、对战消耗、小游戏基础分数、升级所需经验、货币产出比例等；新增卡牌粉尘相关配置，可设置卡牌粉尘提升稀有卡牌获取概率的具体比例、不同稀有度重复卡牌转换粉尘的数量。

2. 怪物与关卡数值：调整单人爬塔各层怪物血量、防御、攻击力、小游戏难度系数、关卡奖励数值，适配游戏平衡性，无需重启服务即可生效。

3. 养成数值配置：修改卡牌升级消耗、强化成功率、共鸣属性加成、小游戏增益效果等养成相关数值，支持数值一键还原功能，防止配置失误。

### 6.7 卡牌导入与管理系统

1. 卡牌信息配置：新增、编辑、删除游戏卡牌，配置卡牌名称、稀有度、属性、技能、伤害倍率、卡牌形象（支持上传webp格式图片，作为卡牌视觉载体）、获取途径、分解材料等完整属性；新增卡牌粉尘转换规则配置，可设置不同稀有度重复卡牌转换为卡牌粉尘的数量；卡牌图片与文字属性分开存储、独立渲染，后期修改卡牌任何文字或数值属性时，无需重新上传或替换卡牌图片，仅修改对应属性值即可实时生效。

2. 批量导入：支持Excel表格批量导入卡牌数据（含卡牌属性信息），卡牌图片需单独上传（支持webp格式），系统自动关联卡牌属性与对应图片；自动校验卡牌信息格式及图片格式，重复卡牌ID提示报错，导入成功后自动同步至游戏卡牌池与图鉴模块，且保持图片与属性分离状态。

3. 卡牌上下线：控制卡牌是否开放获取，下架卡牌标记为绝版，玩家已拥有卡牌保留，未拥有卡牌不再产出，同步更新图鉴展示状态。

4. 卡牌数据备份：支持卡牌配置数据导出备份，防止数据丢失，可通过备份文件快速恢复卡牌配置。

### 6.8 后台数据安全要求

1. 所有敏感操作（数值修改、账号封禁、卡牌删除）需二次确认，防止误操作。

2. 核心数据修改实时同步MySQL数据库，前端游戏端无修改权限，仅做数据展示。

3. 管理员账号定期强制修改密码，异常登录（异地、多次密码错误）自动锁定账号。

## 七、开发优先级调整

1. 第一优先级（基础必备）：游戏端登录注册模块、主界面、人物状态、卡牌包、单人爬塔；后台基础登录、账号管理、游戏基础数值修改

2. 第二优先级（核心补充）：游戏端活动广场、商店、图鉴；后台服务器监控、卡牌导入系统

3. 第三优先级（后期开发）：游戏端实验室、对战模块；后台活动管理、邮箱系统、精细化权限管控

## 八、全局通用需求

1. 适配要求：游戏端优先适配手机端竖屏，CSS响应式布局，支持不同尺寸手机屏幕，操作按钮大小适配手指触摸，无点击偏差；后台管理端适配PC端宽屏，操作界面简洁易用

2. 视觉风格：游戏端统一2D卡通风格，PixiJS渲染卡牌、战场元素，无复杂3D效果，页面简洁无冗余元素；后台端采用Element Plus简约风格，突出功能性，便于快速操作

3. 数据安全：游戏核心数据（卡牌属性、小游戏分数、伤害结算、货币数量）、后台配置数据均由后端MySQL存储与计算，前端仅做展示与操作上传，严格防作弊、防越权

4. 加载要求：游戏端单模块加载时长≤2秒，整体游戏无长时间白屏，加载过程展示极简动画；后台端页面加载时长≤3秒，数据加载快速响应

5. 操作反馈：所有点击、触摸操作均有文字/动效反馈，无操作无响应情况，异常操作给出明确提示

## 九、数据存储补充说明

MySQL数据库新增后台相关存储内容：管理员账号与权限信息、操作日志、服务器监控记录、活动配置数据、邮件数据、全局数值配置、卡牌配置表；新增卡牌粉尘相关存储内容（玩家粉尘数量、粉尘转换规则、粉尘提升概率配置）；原有玩家数据、卡牌数据、闯关对战数据保持不变，前后端所有数据实时同步，保证数据一致性与完整性。
